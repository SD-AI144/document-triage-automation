# document-triage-automation
Rule-based document triage script to scan judgments, flag image-only files for OCR, and output matched keyword metadata to Excel.

An automated document-triage pipeline designed to optimize legal document review. Instead of manually reading dozens of legal judgments to find upload-worthy files, this system runs a targeted rule-based scan to filter and index files automatically.

## How It Works
1. **Keyword Scanning:** Scans document text for predefined pattern signals indicating whether a judgment is upload-worthy or should be discarded.
2. **OCR Flagging:** Detects and flags image-only or scanned documents that require OCR/conversion.
3. **Metadata Indexing:** Outputs a structured Excel sheet containing:
   - Google Drive links to the source documents.
   - Identified matched keywords.
   - Page counts per document.
4. **Triage Review:** Reduces review time by allowing users to check only flagged rows in Excel rather than reading every document from scratch.

## Project Structure
- `document_triage_automation.ipynb`: Google Colab notebook containing the pipeline.
- `data/sample_documents/`: Test files to verify the pipeline.
- `data/sample_output.xlsx`: Example output generated by the script.

## Setup & Usage
1. Open the `document_triage_automation.ipynb` notebook in Google Colab.
2. Follow the steps inside to authenticate Google Drive.
3. Configure your target keyword list inside the config block.
4. Run all cells to process your directory and generate the output Excel file.
